Naming rule for future UI/code discussions
----------------------------------------
Use the visible top/header text first, then include code identifiers in parentheses.
This avoids confusion between what the user sees and what the firmware calls it.

Important clarification:
- BREACH PROTOCOL screen = the former "splash screen".
  Code: STATE_SPLASH / drawSplash() / handleSplashInput()
  File: m5cardputer-cyberpunk-breach-protocol/03_splash_network_menus.ino

- SELECT BOOT NODE = the boot-node selection menu shown inside the BREACH PROTOCOL screen.
  Code: showSplashBootMenu + splashBootFocus inside STATE_SPLASH.
  This is the visible boot menu with HARDWARE NODE, NETWORK NODE, OFFLINE PLAY,
  OTA CATALOG, and SYSTEM SETTINGS.

- Do not call SELECT BOOT NODE the "main menu" unless you explicitly mean the
  user-facing boot selector. In code, STATE_MAIN_MENU is a different screen.

- NETWORK NODE menu = the code-level STATE_MAIN_MENU screen after login/offline entry.
  Code: STATE_MAIN_MENU / drawMainMenu() / handleMainMenuInput()
  Visible header: NETWORK NODE

Input legend used by most screens
---------------------------------
ENTER       Select / confirm
;           Up
.           Down
,           Left / back on some screens
/           Right / description / next on some screens
` / ESC     Back / cancel on most screens
DEL         Delete text, or back on some screens
- / +       Global volume down/up
[ / ]       Global brightness down/up
